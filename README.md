# Playbook de JPGIL

> Un repositorio abierto de guías operativas en Markdown, diseñadas para ser usadas por humanos y agentes IA por igual.

## ¿Qué es un Playbook Personal?

Es un repertorio portátil de buenas prácticas que externaliza el conocimiento operativo de un profesional: cómo nombra variables, cómo estructura commits, qué patrones evita, cómo redacta documentación. Llevas tus prácticas a nuevos proyectos, y lo que aprendes en esos proyectos alimenta de vuelta tu playbook.

👉 [Lee las reglas y la filosofía completa](https://jpgil.github.io/playbook/)

---

## Playbooks

- [agents_good-practices.md](agents_good-practices.md): Cómo configurar agentes IA (`AGENTS.md`) en un repositorio.
- [git-generalidades.md](git-generalidades.md): Convenciones de commits, branches y operaciones Git.
- [markdown_practicas.md](markdown_practicas.md): Convenciones de documentación Markdown y marcadores `#AI-TODO`.
- [github_pages-docsify.md](github_pages-docsify.md): Publicar `/docs/` como sitio Docsify en GitHub Pages.
